# Database conventions

## Naming

| Kind | Pattern | Example |
|---|---|---|
| Table | `snake_case`, plural | `organizations`, `members` |
| Column | `snake_case` | `created_at`, `org_id` |
| Primary key | `id` | – |
| Foreign key | `<referenced_singular>_id` | `org_id`, `user_id` |
| Index | `idx_<table>_<col(s)>` | `idx_members_org_id` |
| Unique | `uq_<table>_<col(s)>` | `uq_members_org_id_user_id` |
| RLS policy | `rls_<table>_<scope>` | `rls_members_tenant_isolation` |
| Enum type | `<scope>_status` | `subscription_status` |

## Tenant model

- Every tenant table has `org_id UUID NOT NULL` + FK to `organizations.id` + index.
- RLS enabled at table create time.
- Policy reads tenant via `current_setting('app.org_id')`.
- App sets `SET LOCAL app.org_id` per query session via `src/server/rls.ts`.

## RLS scope tiers

| Tier | Marker | Example tables |
|---|---|---|
| Tenant | `// tenant-scoped` | `members`, `invitations`, feature tables |
| User-scoped | `// user-scoped` | `sessions`, `accounts` (Better Auth) |
| System | `// system` (no RLS) | `users`, `organizations`, `verification` |

## Timestamps

```ts
created_at: timestamp({ withTimezone: true }).defaultNow().notNull(),
updated_at: timestamp({ withTimezone: true }).defaultNow().notNull().$onUpdate(() => new Date()),
deleted_at: timestamp({ withTimezone: true }), // nullable; partial index
```

Soft-delete: `deleted_at IS NULL` filter at query layer + partial index.

## IDs

PG 18 native: `gen_random_uuid()` (no `uuid-ossp` extension needed).

## Migrations

- Generated by `pnpm db:generate` (drizzle-kit).
- One migration per logical change; never combine schema + data unless atomic by design.
- Hand-edit allowed for: RLS policy refinement, backfill data, `CREATE INDEX CONCURRENTLY`.
- `meta/` is auto-managed: never hand-edit.
- Order for non-trivial column changes: `ALTER → backfill → NOT NULL → ENABLE RLS → CREATE POLICY`.

## What lives outside `db/`

Nothing. Raw SQL outside `db/` and `src/server/rls.ts` is a rule violation ([rules/db.md](../../.claude/rules/db.md)).
