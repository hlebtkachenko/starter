# HANDOFF: hlebtkachenko/flag-external-variants

## Goal

Stop hiding external-registry variants under canonical shadcn slugs. 23 examples sourced from non-shadcn registries (shadcnblocks, reui, shadcnui-blocks) had landed inside `breadcrumb`, `button-group`, `calendar`, `card`, `collapsible`, `combobox`, `command`, `empty` slugs without any deviation marker. Showcase reviewers and AI agents could not tell which were canonical shadcn and which were external imports.

## Current state

- Set `isFlagged: true` on the 23 affected entries in `src/components/examples/_registry-effects.ts`. The Demo wrapper renders a red border + "NOT IN SPEC" pill for flagged entries (already wired in `src/components/showcase/section.tsx`). Verified visually for breadcrumb (5 chips show the pill, default and basic stay card-style).
- New `docs/new-components.md` (1043 lines) generated by `scripts/generate-new-components-doc.ts`. Lists 59 own-slug primitives grouped by `proposedGroup` plus a separate "External variants under shadcn slugs" section with all 23 flagged entries (one row per variant: name, upstream URL, file).
- Helper one-shot `scripts/flag-non-shadcn-variants.ts` left in place for documentation; safe to delete after merge.
- All gates green: typecheck, lint (0 errors), registry build (578 items), orphan check, registry export, dev server visual.

## Decisions made

- Used `isFlagged` rather than minting separate primitive slugs. The flag already exists in the schema for exactly this case ("variant not in upstream `## Examples` block"). Minting fresh slugs would have re-arranged the showcase layout and forced a new registry primitive per imported registry, which is heavier than the deviation actually warrants.
- The doc has two top-level sections: own-slug primitives (rendered under "New Components" band on `/showcase`) and external variants under shadcn slugs (rendered inline with the shadcn primitive, but with the flag pill).
